## Phase Goal

**As a** pustakawan, **I want to** process a book return so the system records the return, automatically fines and blocks the borrower on late returns, and logs the overdue notification, **so that** the loan lifecycle closes correctly and late borrowers are held accountable.

<objective>
Deliver the core return vertical slice end-to-end: a pustakawan opens the new "Sedang Dipinjam" section on `/pinjaman`, clicks "Kembalikan" on a `dipinjam` loan, confirms in a dialog (which previews denda + block for overdue loans), and the backend marks the loan `dikembalikan`, frees the copy, and — when overdue — records `total_denda`, sets `is_diblokir=TRUE`, and logs a stubbed Brevo notification.

This is the first MVP slice. After Task 3, a pustakawan can actually return a book through the UI and observe the fine/block/log effects — a capability that did not exist before.

Purpose: Implements RET-01 (process return), RET-02 (fine + block calculation), RET-03 (Brevo notification stub). Implements locked decisions D-01, D-02, D-03, D-10, D-11.
Output: Extended `peminjaman` router + schema, new `terlambat` StatusBadge variant, new "Sedang Dipinjam" pustakawan section, RED→GREEN backend tests.
</objective>
