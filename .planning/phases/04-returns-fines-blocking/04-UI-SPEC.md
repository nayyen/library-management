## Spacing Scale

Declared values (must be multiples of 4), matching existing `PinjamanPage.jsx` usage:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`, `gap-1.5`) |
| sm | 8px | Compact element spacing (`gap-2`, `py-0.5`) |
| md | 16px | Default cell padding (`px-6`, `py-3`), card padding |
| lg | 24px | Section header padding (`p-6`), grid gaps in blocked-member cards (`gap-4`) |
| xl | 32px | — |
| 2xl | 48px | Stacked-section vertical rhythm (`space-y-8` ≈ 32px; page `space-y-8`) |

Exceptions: none. Buttons use `min-h-[44px]` (existing pattern from Phase 3 "Setujui"/"Tolak"/"Serahkan" — not a spacing-scale value but an accessibility touch-target floor; reuse for "Kembalikan" and "Denda Lunas").

## Typography

Matches existing `DESIGN.md` tokens — already implemented as Tailwind utility classes (`text-{role} font-{role}`). No new sizes/weights introduced in Phase 4.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (body-md) | 16px | 400 (regular) | 24px (1.5) |
| Body small (body-sm) | 14px | 400 (regular) | 20px (≈1.43) |
| Label (label-sm) | 12px | 500 (medium) | 16px (≈1.33) |
| Heading (headline-sm) | 20px | 600 (semibold) | 28px (1.4) |

Phase 4 usage map:
- Section headings ("Sedang Dipinjam", "Anggota Diblokir") → `text-headline-sm font-headline-sm text-primary` (matches "Menunggu Persetujuan"/"Siap Diambil")
- Section subheadings/descriptions → `text-body-sm font-body-sm text-outline`
- Table column headers → `text-label-sm font-label-sm text-outline uppercase tracking-wider`
- Table cell body text (Mahasiswa name, Buku title) → `text-body-md font-body-md`
- Table cell secondary text (penulis, NIM-style subtext, dates) → `text-body-sm font-body-sm text-outline`
- "Denda" column amounts → `text-body-sm font-body-sm font-medium text-alert-crimson` (non-zero) / `text-outline` (the "-" placeholder)
- Blocked-member card name → `text-headline-sm font-headline-sm text-primary` (per `manajemen_anggota` mockup)
- Blocked-member card "Denda Tertunggak" label → `text-body-sm font-body-sm text-secondary` (label) + `text-body-sm font-body-sm font-bold text-alert-crimson` (amount)
- Button labels → `text-label-sm font-label-sm`

## Color

Matches existing 60/30/10 split already established across Phases 1-3. Phase 4 introduces **no new palette colors** — only new *combinations* of existing tokens (crimson badge variant, sage-green unblock button).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `surface-container-lowest` (`#ffffff`) / `surface-container-low` (`#f5f3f5`) | Page background, card/section backgrounds, table row backgrounds (alternating) |
| Secondary (30%) | `primary` / `primary-container` (`#0d1c32`, navy "Ink Blue" family) | Headings (Playfair Display), section titles, "Dipinjam" badge text |
| Accent (10%) | `antique-gold` (`#C5A059`) | Reserved — see list below |
| Destructive | `alert-crimson` (`#93272C`) | "Terlambat" badge, denda amounts, block-related banners/icons, "Tolak" actions (existing) |

**Accent (`antique-gold`) reserved for:** primary CTA buttons elsewhere in the app (e.g. "Ajukan Peminjaman", `EmptyState` action buttons). **Phase 4 introduces no new antique-gold elements** — all Phase 4 action buttons use `sage-green` (positive/unblock) or `ink-blue` (neutral/handover-style) or `alert-crimson` tints (destructive/overdue), consistent with the existing button-color decisions logged in STATE.md (`[Phase 3]` "Serahkan" = ink-blue, "Setujui" = sage-green).

**Second semantic color — `alert-crimson` (`#93272C`) is the destructive/overdue color**, used for:
- "Terlambat" `StatusBadge` variant (badge border + text)
- "Denda" column non-zero amounts (mahasiswa table)
- "Denda Tertunggak" amount on blocked-member cards
- `BlockedBanner` 'blocked' variant (already crimson — unchanged styling, only copy is interpolated)

**`sage-green` (`#8A9A5B`)** — used for the "Denda Lunas" / unblock button (per `manajemen_anggota` mockup's "Buka Blokir" pattern), consistent with "Setujui" being sage-green in Phase 3.

| Error state | Reuses existing pattern: **"Gagal Memuat"** / *"Gagal memuat data pinjaman. Periksa koneksi Anda dan coba lagi."* with "Coba Lagi" retry button (unchanged from Phase 3) |

| Toast — unblock success | **"Denda dinyatakan lunas. Akun {nama_mahasiswa} tidak lagi diblokir."** |
| Toast — generic error | Reuses existing: **"Gagal memproses tindakan. Silakan coba lagi."** |

| Destructive confirmation — Kembalikan (overdue) | ConfirmDialog title: **"Tandai Sudah Dikembalikan?"** · message: **"Buku ini terlambat {X} hari. Denda Rp {X*1000}.000 akan tercatat dan akun mahasiswa akan diblokir."** · confirmLabel: **"Kembalikan"** · `destructive: true` (crimson confirm button signals the block consequence) |
| Destructive confirmation — Denda Lunas | ConfirmDialog title: **"Tandai Denda Lunas?"** · message: **"Denda sebesar Rp {jumlah}.000 milik {nama_mahasiswa} akan dinyatakan lunas dan akun akan dibuka kembali."** · confirmLabel: **"Denda Lunas"** · `destructive: false` (sage-green confirm button — positive/unblocking action) |

**BlockedBanner 'blocked' variant copy template (D-09):**

```
heading: "Akun Diblokir"
body: "Akun Anda diblokir karena denda Rp {jumlah}.000 belum dibayar.
       Selesaikan pembayaran denda di perpustakaan untuk mengajukan
       pinjaman baru."
```

`{jumlah}` = `SUM(total_denda)` across the mahasiswa's `dikembalikan` loans, returned by `GET /api/peminjaman` (e.g., `denda_tertunggak` field on the mahasiswa response root, alongside `is_diblokir`). Format as thousands without decimal — e.g. `5` → "Rp 5.000".

**"Denda" column format (mahasiswa "Pinjaman Saya" table, D-07):**
- `dikembalikan` row with `total_denda > 0` → `"Rp {total_denda formatted}.000"` in `text-alert-crimson font-medium`
- All other rows (including `dikembalikan` with `total_denda == 0`, and any non-`dikembalikan` status) → `"-"` in `text-outline`

Note on Rp formatting: `total_denda` is stored as an integer count of *thousands of Rupiah* equivalent per the existing model convention implied by PRD (`Rp 1.000/day` × days late). Display as `Rp {total_denda}.000` only if `total_denda` is stored as "number of days late" — **confirm with backend schema during planning**: if `total_denda` is stored as the *actual Rupiah amount* (e.g. `5000`), format as `Rp {total_denda.toLocaleString('id-ID')}` (→ "Rp 5.000") instead of string-concatenating ".000". The copy examples above assume the latter (actual Rupiah integer, e.g. `5000`), which is the more defensible schema per PRD §6 ("16 hari terlambat → Rp 2.000")  — **use `Rp {value.toLocaleString('id-ID')}` as the canonical formatter** for all denda displays in this phase (Toast, ConfirmDialog, Denda column, BlockedBanner, blocked-member card).

## Component Specifications

### 1. `StatusBadge.jsx` — new `terlambat` variant

Add to the `variants` map, styled after the `riwayat_peminjaman` mockup's Terlambat badge (`bg-error-container text-on-error-container border-alert-crimson`):

```js
terlambat: {
  label: 'Terlambat',
  class: 'bg-error-container text-on-error-container border-alert-crimson',
  icon: 'warning',
},
```

- Used in place of `dipinjam` badge when `tanggal_tenggat < now()` AND `status_peminjaman === 'dipinjam'` (D-02, D-08).
- Computed client-side per row (no new status enum value) — backend may optionally pre-compute and expose as a derived field (e.g. `is_terlambat: boolean`) per Claude's Discretion in 04-CONTEXT.md; either way the frontend renders `terlambat` instead of `dipinjam` when true.
- Icon choice `warning` (not in existing variant icons) signals urgency, consistent with Material Symbols Outlined usage elsewhere.
- Applies identically for both mahasiswa ("Pinjaman Saya") and pustakawan ("Sedang Dipinjam") tables — same variant, same `compact` prop behavior.

### 2. `BlockedBanner.jsx` — personalized 'blocked' copy (D-09)

Change `variantConfig.blocked.body` from a static string to a function/template that accepts a `dendaAmount` value:

```js
blocked: {
  heading: 'Akun Diblokir',
  body: (dendaAmount) =>
    `Akun Anda diblokir karena denda Rp ${dendaAmount.toLocaleString('id-ID')} belum dibayar. Selesaikan pembayaran denda di perpustakaan untuk mengajukan pinjaman baru.`,
  icon: 'block',
  class: 'bg-alert-crimson/10 border-alert-crimson/30 text-alert-crimson',
  iconClass: 'text-alert-crimson',
},
```

- Component signature becomes `BlockedBanner({ variant, dendaAmount })`. When `variant === 'blocked'`, render `config.body(dendaAmount ?? 0)`; for `variant === 'limit'`, `config.body` remains a plain string (no breaking change to that branch — guard with `typeof config.body === 'function'`).
- `dendaAmount` is sourced from the new `denda_tertunggak` (or equivalent) field on the mahasiswa's `GET /api/peminjaman` response root.
- Visual styling (crimson background/border/icon, `block` icon, layout, `role="status" aria-live="polite"`) is **unchanged** from Phase 3.

### 3. New pustakawan section: "Sedang Dipinjam" (3rd stacked section, D-01)

Placement: after "Siap Diambil", before "Anggota Diblokir". Same `<section>` shell as existing sections:

```
bg-surface-container-lowest border border-paper-shadow rounded-xl overflow-hidden
```

Header block (matches "Siap Diambil" pattern):
- Title: `text-headline-sm font-headline-sm text-primary` → **"Sedang Dipinjam"**
- Subtitle: `text-body-sm font-body-sm text-outline mt-1` → **"Semua buku yang sedang dipinjam, diurutkan berdasarkan yang paling terlambat."**

Table columns (left to right):

| Column | Header (label-sm uppercase) | Cell content |
|--------|------|---------------|
| Mahasiswa | "Mahasiswa" | Avatar circle (initial) + name + "Mahasiswa" subtext — identical markup to "Menunggu Persetujuan"/"Siap Diambil" mahasiswa cell |
| Buku | "Buku" | `BookCell` (existing, reused as-is) |
| Tenggat | "Tenggat" | `formatDate(item.tanggal_tenggat)`. If overdue, prefix/suffix with crimson styling: `<span className="text-alert-crimson font-medium">{date}</span>` |
| Status | "Status" | `StatusBadge` — `terlambat` if overdue else `dipinjam` |
| Aksi | "Aksi" | "Kembalikan" button |

Sort order: rows sorted by `tanggal_tenggat` ascending (most overdue first) — implemented server-side per D-01, frontend renders in the order received.

**"Kembalikan" button:**
```jsx
<button
  type="button"
  onClick={() => handleKembalikan(item)}
  className="inline-flex items-center gap-1.5 px-4 py-2 rounded-full text-label-sm font-label-sm text-white bg-ink-blue hover:opacity-90 transition-opacity min-h-[44px]"
  aria-label={`Tandai buku ${item.judul} sudah dikembalikan`}
>
  <span className="material-symbols-outlined text-[18px]">assignment_return</span>
  Kembalikan
</button>
```
Color: `bg-ink-blue` — same neutral-action color as "Serahkan" (Phase 3 D-14 precedent: ink-blue for the second-tier neutral pustakawan action; sage-green reserved here for "Denda Lunas" which is the explicitly positive/unblocking action).

**ConfirmDialog payload** (via existing `showConfirm` helper):
- On-time (`tanggal_tenggat >= now`):
  - title: "Tandai Sudah Dikembalikan?"
  - message: `Tandai buku ini sudah dikembalikan?`
  - confirmLabel: "Kembalikan", `destructive: false`
- Overdue (`tanggal_tenggat < now`):
  - title: "Tandai Sudah Dikembalikan?"
  - message: `Buku "${item.judul}" terlambat ${X} hari. Denda Rp ${(X*1000).toLocaleString('id-ID')} akan tercatat dan akun ${item.nama_mahasiswa} akan diblokir.` where `X = Math.floor((now - tanggal_tenggat) / 86400000)` (client-side estimate per D-03; server recalculates authoritatively on confirm)
  - confirmLabel: "Kembalikan", `destructive: true` (crimson button — signals the blocking consequence)

On success: `setToast({ type: 'success', message })` using the on-time/late toast copy above (use the server's authoritative `total_denda`/`is_diblokir` from the response, not the client estimate), then `setRefreshKey((k) => k + 1)`.

Empty state: `<EmptyState icon="inventory_2" title="Tidak Ada Pinjaman Aktif" message="Tidak ada buku yang sedang dipinjam saat ini." />` (no action button).

### 4. New pustakawan section: "Anggota Diblokir" (4th stacked section, D-04)

Placement: last section, after "Sedang Dipinjam". Same `<section>` shell.

Header block:
- Title: `text-headline-sm font-headline-sm text-primary` → **"Anggota Diblokir"**
- Subtitle: `text-body-sm font-body-sm text-outline mt-1` → **"Anggota dengan denda tertunggak — buka blokir setelah denda dibayar."**

**Layout: card list (not table)** — follows the `manajemen_anggota` mockup's blocked-member card pattern, since each entry needs more visual weight (name, email/identifier, denda amount, action) than a table row. Cards stack vertically with `space-y-4`, each card padded `p-6` inside the section body (`p-6` wrapper, not table `overflow-x-auto`).

Per-card markup (adapted from `manajemen_anggota_biblio/code.html` "Member Card 2"):

```jsx
<div className="bg-surface-container-lowest border border-alert-crimson/30 rounded-xl p-4 sm:p-6 flex flex-col sm:flex-row sm:items-center sm:justify-between gap-4">
  <div className="flex items-center gap-4">
    <div className="w-12 h-12 bg-alert-crimson/10 text-alert-crimson rounded-full flex items-center justify-center text-headline-sm font-headline-sm font-bold shrink-0">
      {initial}
    </div>
    <div>
      <h3 className="text-headline-sm font-headline-sm text-primary">{nama_mahasiswa}</h3>
      <p className="text-label-sm font-label-sm text-secondary">{email}</p>
    </div>
  </div>
  <div className="flex items-center justify-between sm:justify-end gap-6">
    <div>
      <p className="text-body-sm font-body-sm text-secondary mb-1">Denda Tertunggak</p>
      <p className="text-body-sm font-body-sm font-bold text-alert-crimson">
        Rp {total.toLocaleString('id-ID')}
      </p>
    </div>
    <button
      type="button"
      onClick={() => handleLunasiDenda(item)}
      className="inline-flex items-center gap-1.5 px-4 py-2 rounded-full text-label-sm font-label-sm bg-sage-green text-on-primary hover:opacity-90 transition-opacity min-h-[44px]"
      aria-label={`Tandai denda ${nama_mahasiswa} lunas`}
    >
      <span className="material-symbols-outlined text-[18px]">lock_open</span>
      Denda Lunas
    </button>
  </div>
</div>
```

- Card border uses `border-alert-crimson/30` (matches mockup's blocked-card emphasis), distinguishing it from the neutral `border-paper-shadow` used by table-based sections.
- "Denda Tertunggak" value = `SUM(total_denda)` across the mahasiswa's `dikembalikan` loans (D-06), formatted via `Rp {value.toLocaleString('id-ID')}`.
- "Denda Lunas" button: `bg-sage-green text-on-primary`, icon `lock_open`, label "Denda Lunas" — combines PRD literal wording with the mockup's "Buka Blokir" sage-green/`lock_open` styling (per Claude's Discretion note in 04-CONTEXT.md).

**ConfirmDialog payload:**
- title: "Tandai Denda Lunas?"
- message: `Denda sebesar Rp ${total.toLocaleString('id-ID')} milik ${nama_mahasiswa} akan dinyatakan lunas dan akun akan dibuka kembali.`
- confirmLabel: "Denda Lunas", `destructive: false` (sage-green confirm button per `ConfirmDialog`'s `destructive` prop — `false` renders `bg-antique-gold`; **note**: existing `ConfirmDialog` only supports crimson/gold for confirm button, not sage-green — see Open Question below)

On success: `setToast({ type: 'success', message: 'Denda dinyatakan lunas. Akun {nama} tidak lagi diblokir.' })`, then `setRefreshKey((k) => k + 1)`.

Empty state: `<EmptyState icon="task_alt" title="Tidak Ada Anggota Diblokir" message="Semua anggota dalam status baik — tidak ada denda tertunggak." />` (no action button).

**Open question for planner/executor:** `ConfirmDialog`'s `destructive` prop only toggles between `bg-alert-crimson` (true) and `bg-antique-gold` (false) for the confirm button — neither matches `sage-green`. For "Denda Lunas" and the on-time "Kembalikan" confirm, the **antique-gold fallback (`destructive: false`) is acceptable** since `ConfirmDialog` is a generic primitive not meant to carry every action color; the *page-level button* (sage-green "Denda Lunas", ink-blue "Kembalikan") carries the correct color, and the dialog's confirm button can remain antique-gold/crimson per its existing two-tone contract. Do not modify `ConfirmDialog.jsx` to add a third color variant — out of scope for Phase 4.
