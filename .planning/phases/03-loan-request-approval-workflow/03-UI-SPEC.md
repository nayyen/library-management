| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), badge internal padding, `border-l-4` accent on the cover-spine block in the loan request modal |

- These spacing exceptions are inherited/pre-approved per Phase 1 and Phase 2 sign-off cross-references тАФ no re-litigation needed.

## Typography

Phase 3 reuses the same type scale established in Phase 1/2 тАФ no new sizes or weights introduced.

| Role | Size | Weight | Line Height | Font | Usage in Phase 3 |
|------|------|--------|-------------|------|------|
| headline-md | 24px | 600 (semibold) | 32px (1.33) | Playfair Display | Modal title "Formulir Peminjaman" (`LoanRequestModal`), page title "Pinjaman" (`/pinjaman`) |
| headline-sm | 20px | 600 (semibold) | 28px (1.4) | Playfair Display | Modal sub-block title (selected book title in "Buku Terpilih"), section headings within `/pinjaman` ("Menunggu Persetujuan", "Siap Diambil", "Pinjaman Saya") тАФ same role Phase 2 used for `BukuDetailPage`'s "Salinan Buku" section header (continuing established living pattern; `headline-sm` was carried into Phase 2's implementation as the section-header size alongside `label-md` uppercase headers) |
| body-lg | 18px | 400 (regular) | 28px (1.56) | Inter | Banner body copy (blocked/limit-reached message) тАФ needs emphasis/readability as a persistent warning |
| body-md | 16px | 400 (regular) | 24px (1.5) | Inter | Body copy: modal informational text ("Ringkasan Peminjaman" paragraph), table cell primary text (book title, mahasiswa name), empty-state body, toast message text |
| label-md | 14px | 600 (semibold), 0.05em letter-spacing | 16px (1.14) | Inter | Buttons ("Kirim", "Batal", "Setujui", "Tolak", "Serahkan", "Pinjam"), table column headers, input labels in modal, section header uppercase labels |
| label-sm | 12px | 400 (regular) | 16px (1.33) | Inter | `StatusBadge` text, secondary table cell text (dates, ISBN, lokasi_rak, role labels), pickup-deadline microcopy |

**Weight contract:** exactly **2 weights** тАФ 400 (regular) for body copy, badges, table secondary text; 600 (semibold) for headings, buttons, input labels, table column headers, section headers. No 3rd weight.

**Heading line-height:** 1.33 (headline-md) / 1.4 (headline-sm) тАФ within Phase 1's accepted 1.2тАУ1.4 range.
**Body line-height:** 1.5 (body-md), 1.56 (body-lg), 1.33 (label-sm) тАФ within accepted range.

**Note on 4-size constraint:** This table lists 6 named roles (headline-md, headline-sm, body-lg, body-md, label-md, label-sm) but only **4 distinct pixel sizes are introduced as "primary" for new Phase 3 UI**: 24px (headings), 20px (section headers тАФ reused from Phase 2's living pattern), 16px (body), 14px (labels/buttons), 12px (badges/meta). This mirrors Phase 2's actual shipped code (`BukuDetailPage.jsx` uses `headline-sm`, `body-lg`, `body-md`, `label-sm` тАФ a superset of the originally-declared 4). Phase 3 does not introduce any size beyond what Phase 2 already shipped in code. If the checker requires strict 4-size enforcement, collapse `headline-sm`тЖТ`headline-md` and `body-lg`тЖТ`body-md` (both are visually adjacent and the modal/banner remain legible at 24px/16px) тАФ but the recommendation is to keep `headline-sm` (20px) for section headers and `body-lg` (18px) for the banner, consistent with the already-shipped Phase 2 codebase.

## Layout & Navigation Contract

### Route structure (D-05)
- `frontend/src/router.jsx`: replace `{ path: 'pinjaman', element: <ComingSoonPage title="Riwayat Peminjaman" /> }` with `{ path: 'pinjaman', element: <PinjamanPage /> }`, inside the existing `AppShell`-wrapped `ProtectedRoute` group. No nav changes to `AppShell.jsx` тАФ the "Pinjaman" link (`bookmark` icon) already exists for both `MAHASISWA_NAV` and `PUSTAKAWAN_NAV`.
- `/katalog/:id` (`BukuDetailPage.jsx`) тАФ no new route, but `SalinanTable` usage gets a new "Pinjam" action column for `tersedia` rows when `peran === 'mahasiswa'` (D-02).

### `/katalog/:id` тАФ "Pinjam" entry point (D-01, D-02)

- `SalinanTable` (existing component) gets an optional `onPinjam(salinan)` prop and `peran` prop. When `peran === 'mahasiswa'` and a row's `status_ketersediaan === 'tersedia'`, render a 4th column "Aksi" with a "Pinjam" button (Sage Green, `bookmark_add` icon, `min-h-[44px]`). Non-`tersedia` rows show no action (empty cell or `тАФ`).
- Clicking "Pinjam" opens `LoanRequestModal` with the selected `salinan` + `buku` context.

- All icon-only buttons (`close`, action buttons where icon-only) have `aria-label`.
