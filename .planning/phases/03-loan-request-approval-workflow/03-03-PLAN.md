## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (pustakawan) → API | Pustakawan-only mutations; role re-enforced server-side regardless of which UI branch rendered |
| rendered API data → DOM | Mahasiswa names / book titles rendered into the queue tables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Elevation of Privilege | a mahasiswa reaching the pustakawan UI branch | mitigate | The pustakawan branch only renders when the decoded `peran === 'pustakawan'`; even if forced, the persetujuan/serahkan endpoints enforce `_pustakawan_only` server-side (403, Plan 01 T-03-01). UI branch is presentation only. |
| T-03-12 | Tampering | accidental/duplicate state transitions | mitigate | Every action requires a ConfirmDialog (D-07) before the API call; the API rejects invalid-state transitions with 409 (Plan 01 T-03-06). actionLoading disables the confirm button during the in-flight request to prevent double-submit. |
| T-03-13 | Information Disclosure | XSS via mahasiswa name / book title in queue cells | mitigate | React escapes string children; no `dangerouslySetInnerHTML`. Avatar initials derived from the name string, not injected as HTML. |
| T-03-SC | Tampering | npm installs | accept | No new npm packages — AntrianTable is hand-built React + Tailwind reusing existing components and Material Symbols. No package-manager installs in this plan. |
</threat_model>
