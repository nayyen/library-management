**Source:** `01-RESEARCH.md` → "Pattern 2: pwdlib Argon2 hashing" (lines ~295-313)
\`\`\`python
from pwdlib import PasswordHash
password_hash = PasswordHash.recommended()
def get_password_hash(password: str) -> str:
    return password_hash.hash(password)
def verify_password(plain_password: str, hashed_password: str) -> bool:
    return password_hash.verify(plain_password, hashed_password)
\`\`\`

## Shared Patterns

### Authentication / Authorization (backend)
**Source:** RESEARCH.md Pattern 1 (`get_current_user` + `require_role`)
**Apply to:** Every protected route added in this phase and ALL future phases (catalog, borrowing, loans, notifications)
\`\`\`python
def require_role(*allowed_roles: UserRole):
    async def _checker(user: User = Depends(get_current_user)) -> User:
        if user.role not in allowed_roles:
            raise HTTPException(status_code=403, detail="You do not have permission to perform this action")
        return user
    return _checker
\`\`\`

### Error handling (backend)
**Source:** RESEARCH.md Pattern 1 (`credentials_exception` shape) + Pattern 4 (`HTTPException(401, ...)` for token errors)
**Apply to:** All router and service files — use `HTTPException` with explicit status codes and `WWW-Authenticate` headers where relevant; never leak whether a credential exists vs. is wrong (login should return one generic "Invalid email or password", per the Security Domain table's STRIDE note on enumeration).

### CORS + cookie configuration
**Source:** RESEARCH.md Pitfalls 1 & 2 (lines ~574-586)
**Apply to:** `backend/app/main.py` CORS middleware setup and `frontend/vite.config.ts`
- Never combine `allow_credentials=True` with wildcard `allow_origins=["*"]`
- Pick ONE hostname (`localhost`) consistently across Vite config, CORS `allow_origins`, cookie domain (don't set explicit `domain=`), and `VITE_API_URL`

### Response/session formatting (frontend)
**Source:** RESEARCH.md "Zustand auth store" + Pattern 7
**Apply to:** All auth-related hooks (`useLogin`, `useSilentRefresh`) — store `accessToken` + `user` in Zustand only (never localStorage), let TanStack Query own server-state/loading/error, axios interceptor owns token attachment + refresh-on-401.

### Validation
**Source:** RESEARCH.md "Don't Hand-Roll" table — Pydantic schemas (backend), Zod + react-hook-form (frontend)
**Apply to:** `backend/app/schemas/auth.py`/`user.py` (use `EmailStr`, password length/complexity constraints, conditional `librarian_code` requirement when `role == "librarian"`) and frontend form pages (`LoginPage`, `SignupPage`, etc.)
