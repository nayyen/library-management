## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → backend (CORS, cross-port dev) | Untrusted origin; httpOnly refresh cookie crosses here. Misconfigured CORS/cookie = silent session failure or credential leak. |
| backend → PostgreSQL | App-to-DB; connection string carries credentials (must come from gitignored `.env`, never the compose file body). |
| developer machine → Docker daemon | Stack runs as containers; secrets in `.env` must not enter version control. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-CORS | Spoofing/Info Disclosure | main.py CORS middleware | mitigate | Explicit `allow_origins=["http://localhost:5173"]` + `allow_credentials=True`; NEVER wildcard with credentials (RESEARCH Pitfall 2). Acceptance check greps for this. |
| T-01-SECRET | Information Disclosure | docker-compose.yml / .env | mitigate | `SECRET_KEY`, DB creds, `LIBRARIAN_SIGNUP_CODE` live only in gitignored `.env` (loaded via env_file + pydantic-settings); `.env.example` ships placeholders only; `.gitignore` includes `.env`. |
| T-01-COOKIE-HOST | Spoofing | cookie/CORS host mismatch | mitigate | One hostname (`localhost`) across Vite, CORS, cookie (no explicit domain), VITE_API_URL (RESEARCH Pitfall 1). |
| T-01-SC | Tampering | npm/uv installs (pwdlib, fastapi-mail) | mitigate | pwdlib + fastapi-mail are [ASSUMED] per RESEARCH Package Legitimacy Audit (slopcheck unavailable). Both are named by CLAUDE.md / official FastAPI PR #13917. Lightweight gate: install then confirm `from pwdlib import PasswordHash` imports cleanly (covered when Plan 02 first imports them). No [SLOP]/[SUS] packages present. |
