<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /auth/* | Untrusted credentials, tokens, and the role/librarian_code claim cross here. |
| access token (client-held) → require_role | The token's `role` claim must be re-verified server-side every request; UI claims are never trusted (AUTH-04). |
| refresh cookie → /auth/refresh | A bearer credential; theft or replay must be detectable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing / Elevation | signup role + librarian_code (AUTH-01) | mitigate | Server compares `librarian_code` to `LIBRARIAN_SIGNUP_CODE` (env); wrong/missing → 400, NO silent student fallback (D-03). Tested by test_signup_librarian_invalid_code. |
| T-01-02 | Spoofing / Elevation | refresh-token theft + replay (AUTH-02) | mitigate | httpOnly+SameSite=Lax cookie (D-05); rotation-on-use under `SELECT … FOR UPDATE` (Pattern 4); reuse detection revokes the whole session family. Tested by test_refresh_rotates_token. |
| T-01-04 | Elevation of Privilege | wrong-role access to librarian route (AUTH-04) | mitigate | `require_role` re-checks the DB-loaded user's role server-side → 403; UI gating is cosmetic only. Tested by test_require_role_rejects_wrong_role + manual forged-token check (01-VALIDATION). |
| T-01-ALGCONF | Tampering | JWT `alg:none` / HS↔RS confusion | mitigate | Every `jwt.decode` passes explicit `algorithms=["HS256"]`; algorithm never read from token header. Acceptance check greps for the explicit list. |
| T-01-ENUM | Information Disclosure | login error messages | mitigate | Single generic "Invalid email or password" 401 for both bad-email and bad-password (Security Domain). |
| T-01-PWHASH | Information Disclosure | password storage | mitigate | Argon2id via `pwdlib.PasswordHash.recommended()`; refresh tokens stored as SHA-256 hashes, never raw. |
| T-01-BRUTE | Denial of Service / Spoofing | unthrottled login/signup | accept | Rate-limiting NOT locked in CONTEXT.md; deferred per SKELETON "Out of Scope" (LOW value for an internal tool; add slowapi in a hardening pass). Flagged, not silently dropped. |

**HIGH-severity gate:** T-01-04 (role bypass) and T-01-02 (refresh theft/replay) are the highest-severity threats; both are mitigated with explicit tested controls in this plan. No HIGH threat is left at `accept`.
