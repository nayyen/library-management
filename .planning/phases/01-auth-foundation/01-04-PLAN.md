<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /auth/forgot-password | Untrusted email; response must not reveal whether it is registered (D-09). |
| reset link (email) → /auth/reset-password | A bearer capability; must be single-use, short-lived, hashed-at-rest. |
| reset completion → existing sessions | The recovery moment; a potentially-compromised account's old sessions (incl. an attacker's) must all be invalidated (D-07). |
| backend → Mailpit SMTP | Dev-only; no credentials/TLS. Prod swaps the SMTP target via settings (not in scope). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-ENUM | Information Disclosure | forgot-password response | mitigate | Identical generic response both branches; unconditional user lookup (D-09). Tested by test_forgot_password_enumeration_safe. |
| T-01-03-FIX | Elevation of Privilege | session fixation after reset | mitigate | reset_password calls revoke_all_user_sessions → exactly one valid session post-reset (D-07). Tested by test_reset_revokes_all_sessions. |
| T-01-03-REPLAY | Spoofing | reset-link reuse/expiry | mitigate | Single-use (`used_at`) + 1-hour expiry; hashed-at-rest token (D-08). Tested by test_reset_password_single_use. |
| T-01-03-SMTPBLOCK | Denial of Service | synchronous SMTP in request | mitigate | `BackgroundTasks.add_task` fire-and-forget; HTTP response never waits on SMTP (CLAUDE.md / RESEARCH anti-pattern). Acceptance greps for add_task. |
| T-01-03-TIMING | Information Disclosure | timing side-channel on forgot-password | accept | Sub-ms timing-attack equalization NOT implemented (RESEARCH A1, LOW risk for an internal tool); the obvious message-text vector IS closed. Flagged in SKELETON Out-of-Scope. |

**HIGH-severity gate:** session fixation (T-01-03-FIX) is the highest-severity reset threat and is mitigated + tested. No HIGH threat left at `accept` (the lone `accept` is a LOW-risk timing side-channel).
</threat_model>
